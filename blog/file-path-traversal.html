File Path Traversal, Simple Cases

Date: 2025-06-09 | Tags: Web Security, Tutorials

File path traversal, also known as directory traversal, is a vulnerability that allows attackers to access files outside the intended directory. In simple cases, this can be exploited by using sequences like '../' in input fields.

Example: If a web app loads files via 'load.php?file=user.txt', an attacker might try 'load.php?file=../../etc/passwd' to access sensitive system files.

Mitigation: Sanitize user input, use whitelists for file names, and avoid direct file path construction from inputs.
